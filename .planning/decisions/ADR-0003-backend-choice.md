# ADR-0003: 백엔드 및 AI 서비스 선택

- **날짜**: 2026-06-01
- **상태**: 확정

---

## 배경

일기 데이터를 저장하고, AI가 감정 분석을 수행하는 구조가 필요하다.
별도 서버를 구축하거나, 클라우드 AI API를 직접 호출하는 방식 중 선택해야 했다.

## 선택지 — 데이터 저장

| 방식 | 장점 | 단점 |
|------|------|------|
| **로컬 저장 (SharedPreferences)** | 서버 불필요, 개인정보 기기 내 보관 | 기기 분실 시 데이터 소실 |
| Firebase Firestore | 실시간 동기화, 멀티 기기 지원 | 설정 복잡, 비용 발생 가능 |
| 자체 서버 (REST API) | 완전한 제어권 | 개발/운영 비용 높음 |

## 선택지 — AI 서비스

| 서비스 | 장점 | 단점 |
|--------|------|------|
| **Google Gemini API** | 무료 쿼터 넉넉, 한국어 품질 우수 | 간헐적 429 오류 |
| OpenAI GPT-4o | 높은 품질 | 유료, 비용 부담 |
| Claude (Anthropic) | 공감 능력 우수 | 무료 쿼터 제한적 |
| 온디바이스 모델 | 오프라인 동작 | 품질 낮음, 용량 큼 |

## 결정

- **저장**: 로컬 (`SharedPreferences`) — 서버 없는 구조
- **AI**: `Google Gemini 1.5 Flash` API 직접 호출

## 이유

**저장 — 로컬 선택 이유**
- 일기는 매우 개인적인 데이터 → 서버 전송 최소화가 바람직
- 수업 범위에서 Firebase 설정 복잡도를 줄이기 위함
- SharedPreferences로 JSON 직렬화하여 충분히 구현 가능

**AI — Gemini 1.5 Flash 선택 이유**
- 무료 쿼터(15 RPM, 1,500 RPD)로 개발 및 시연에 충분
- 한국어 감정 분석 품질이 실사용 수준
- `gemini-2.0-flash` 시도 → 429 오류 발생 → `gemini-1.5-flash`로 변경
- REST API 직접 호출로 별도 SDK 불필요 (dio 사용)

## 결과

- 일기 데이터: 기기 로컬에만 저장, 서버 전송 없음
- AI 분석: 일기 내용만 Gemini API로 전송 (서버에 일기 전체 저장 안 함)
- API 키: `.env` 파일 관리, `.gitignore` 처리로 코드 노출 방지
