# 요구사항

## 기능적 요구사항 (Functional Requirements)

### 일기 관리
| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| F-01 | 사용자는 날짜를 선택하여 일기를 작성할 수 있다 | 필수 |
| F-02 | 작성한 일기는 기기 로컬에 저장된다 | 필수 |
| F-03 | 저장된 일기를 수정할 수 있다 | 필수 |
| F-04 | 저장된 일기를 삭제할 수 있다 | 필수 |
| F-05 | 일기 목록을 월별 캘린더 형태로 볼 수 있다 | 필수 |

### AI 분석
| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| F-06 | 일기 저장 시 AI가 자동으로 감정을 분석한다 | 필수 |
| F-07 | AI는 감정을 이모지 하나로 표현한다 | 필수 |
| F-08 | AI는 2~3문장의 공감 조언 코멘트를 생성한다 | 필수 |
| F-09 | AI 분석 중에는 로딩 상태를 표시한다 | 필수 |

### 감정 시각화
| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| F-10 | 캘린더의 날짜에 해당 날 감정 이모지가 표시된다 | 필수 |
| F-11 | 일기 목록 카드에 감정 이모지와 내용 미리보기가 표시된다 | 필수 |

## 비기능적 요구사항 (Non-Functional Requirements)

| ID | 요구사항 | 기준 |
|----|----------|------|
| NF-01 | 앱은 Android / iOS 크로스플랫폼으로 동작한다 | Flutter 사용 |
| NF-02 | AI 코멘트는 10초 이내에 생성된다 | 네트워크 정상 환경 기준 |
| NF-03 | 일기 데이터는 로컬에만 저장된다 (개인정보 보호) | SharedPreferences 사용 |
| NF-04 | API 키는 .env 파일로 관리하며 코드에 노출되지 않는다 | 보안 |

## 기술 스택

| 분류 | 기술 |
|------|------|
| 프레임워크 | Flutter (Dart) |
| 상태 관리 | flutter_riverpod |
| 라우팅 | go_router |
| 로컬 저장 | shared_preferences |
| AI API | Google Gemini 1.5 Flash |
| HTTP | dio |
| 캘린더 UI | table_calendar |
