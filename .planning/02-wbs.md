# Work Breakdown Structure (WBS)

## 전체 구조

```
AI 감정 일기 앱
├── 1. 기획
│   ├── 1.1 아이디어 정의 및 비전 작성
│   ├── 1.2 요구사항 분석
│   └── 1.3 WBS 및 일정 작성
│
├── 2. 설계
│   ├── 2.1 화면 구성 설계 (3개 스크린)
│   ├── 2.2 데이터 모델 설계 (DiaryEntry)
│   ├── 2.3 상태 관리 설계 (Riverpod)
│   └── 2.4 API 연동 설계 (Gemini)
│
├── 3. 개발
│   ├── 3.1 프로젝트 초기 세팅
│   │   ├── 3.1.1 Flutter 프로젝트 생성
│   │   ├── 3.1.2 패키지 의존성 설정
│   │   └── 3.1.3 폴더 구조 세팅
│   │
│   ├── 3.2 데이터 레이어
│   │   ├── 3.2.1 DiaryEntry 모델
│   │   └── 3.2.2 DiaryRepository (로컬 저장)
│   │
│   ├── 3.3 AI 서비스
│   │   ├── 3.3.1 GeminiService 구현
│   │   └── 3.3.2 AI Provider (Riverpod)
│   │
│   └── 3.4 UI
│       ├── 3.4.1 홈 화면 (캘린더 + 목록)
│       ├── 3.4.2 일기 작성 화면
│       └── 3.4.3 일기 상세 + AI 코멘트 화면
│
├── 4. 테스트
│   ├── 4.1 기능 테스트 (일기 CRUD)
│   ├── 4.2 AI 연동 테스트
│   └── 4.3 기기 테스트 (Android)
│
└── 5. 마무리
    ├── 5.1 버그 수정
    ├── 5.2 문서 정리
    └── 5.3 최종 제출
```
